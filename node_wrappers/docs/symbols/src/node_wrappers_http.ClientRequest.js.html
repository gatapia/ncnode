  4 /**
  5  * @fileoverview This object is created internally and returned from `http.request()`.  It
  6  * represents an _in-progress_ request whose header has already been sent.
  7  *
  8  * To get the response, add a listener for `'response'` to the request object.
  9  * `'response'` will be emitted from the request object when the response
 10  * headers have been received.  The `'response'` event is executed with one
 11  * argument which is an instance of `http.ClientResponse`.
 12  *
 13  * During the `'response'` event, one can add listeners to the
 14  * response object; particularly to listen for the `'data'` event. Note that
 15  * the `'response'` event is called before any part of the response body is received,
 16  * so there is no need to worry about racing to catch the first part of the
 17  * body. As long as a listener for `'data'` is added during the `'response'`
 18  * event, the entire body will be caught.
 19  *
 20  * <pre>
 21  *     // Good
 22  *     request.on('response', function (response) {
 23  *       response.on('data', function (chunk) {
 24  *         console.log('BODY: ' + chunk);
 25  *       });
 26  *     });
 27  *
 28  *     // Bad - misses all or part of the body
 29  *     request.on('response', function (response) {
 30  *       setTimeout(function () {
 31  *         response.on('data', function (chunk) {
 32  *           console.log('BODY: ' + chunk);
 33  *         });
 34  *       }, 10);
 35  *     });
 36  * </pre>
 37  * This is a `Writable Stream`.
 38  *
 39  * This is an `EventEmitter` with the following events:
 40  */
