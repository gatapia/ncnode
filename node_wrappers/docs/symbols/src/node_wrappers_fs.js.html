  4 /**
  5  * @fileoverview File I/O is provided by simple wrappers around standard POSIX functions.  To
  6  * use this module do `require('fs')`. All the methods have asynchronous and
  7  * synchronous forms.
  8  *
  9  * The asynchronous form always take a completion callback as its last argument.
 10  * The arguments passed to the completion callback depend on the method, but the
 11  * first argument is always reserved for an exception. If the operation was
 12  * completed successfully, then the first argument will be `null` or `undefined`.
 13  *
 14  * Here is an example of the asynchronous version:
 15  * <pre>
 16  *     var fs = require('fs');
 17  *
 18  *     fs.unlink('/tmp/hello', function (err) {
 19  *       if (err) throw err;
 20  *       console.log('successfully deleted /tmp/hello');
 21  *     });
 22  * </pre>
 23  * Here is the synchronous version:
 24  * <pre>
 25  *     var fs = require('fs');
 26  *
 27  *     fs.unlinkSync('/tmp/hello')
 28  *     console.log('successfully deleted /tmp/hello');
 29  * </pre>
 30  * With the asynchronous methods there is no guaranteed ordering. So the
 31  * following is prone to error:
 32  * <pre>
 33  *     fs.rename('/tmp/hello', '/tmp/world', function (err) {
 34  *       if (err) throw err;
 35  *       console.log('renamed complete');
 36  *     });
 37  *     fs.stat('/tmp/world', function (err, stats) {
 38  *       if (err) throw err;
 39  *       console.log('stats: ' + JSON.stringify(stats));
 40  *     });
 41  * </pre>
 42  * It could be that `fs.stat` is executed before `fs.rename`.
 43  * The correct way to do this is to chain the callbacks.
 44  * <pre>
 45  *     fs.rename('/tmp/hello', '/tmp/world', function (err) {
 46  *       if (err) throw err;
 47  *       fs.stat('/tmp/world', function (err, stats) {
 48  *         if (err) throw err;
 49  *         console.log('stats: ' + JSON.stringify(stats));
 50  *       });
 51  *     });
 52  * </pre>
 53  * In busy processes, the programmer is _strongly encouraged_ to use the
 54  * asynchronous versions of these calls. The synchronous versions will block
 55  * the entire process until they complete--halting all connections.
 56  */
 57
