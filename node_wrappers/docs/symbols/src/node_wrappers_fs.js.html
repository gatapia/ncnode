1 /**
  2  * @name node.fs
  3  * @namespace
  4  * File I/O is provided by simple wrappers around standard POSIX functions.  To
  5  * use this module do <code class="code prettyprint lang-js">require('fs')</code>. All the methods have asynchronous and
  6  * synchronous forms.
  7  *
  8  * The asynchronous form always take a completion callback as its last argument.
  9  * The arguments passed to the completion callback depend on the method, but the
 10  * first argument is always reserved for an exception. If the operation was
 11  * completed successfully, then the first argument will be <code class="code prettyprint lang-js">null</code> or <code class="code prettyprint lang-js">undefined</code>.
 12  *
 13  * Here is an example of the asynchronous version:
 14  * <pre class="code prettyprint lang-js">
 15  *     var fs = require('fs');
 16  *
 17  *     fs.unlink('/tmp/hello', function (err) {
 18  *       if (err) throw err;
 19  *       console.log('successfully deleted /tmp/hello');
 20  *     });
 21  * </pre>
 22  * Here is the synchronous version:
 23  * <pre class="code prettyprint lang-js">
 24  *     var fs = require('fs');
 25  *
 26  *     fs.unlinkSync('/tmp/hello')
 27  *     console.log('successfully deleted /tmp/hello');
 28  * </pre>
 29  * With the asynchronous methods there is no guaranteed ordering. So the
 30  * following is prone to error:
 31  * <pre class="code prettyprint lang-js">
 32  *     fs.rename('/tmp/hello', '/tmp/world', function (err) {
 33  *       if (err) throw err;
 34  *       console.log('renamed complete');
 35  *     });
 36  *     fs.stat('/tmp/world', function (err, stats) {
 37  *       if (err) throw err;
 38  *       console.log('stats: ' + JSON.stringify(stats));
 39  *     });
 40  * </pre>
 41  * It could be that <code class="code prettyprint lang-js">fs.stat</code> is executed before <code class="code prettyprint lang-js">fs.rename</code>.
 42  * The correct way to do this is to chain the callbacks.
 43  * <pre class="code prettyprint lang-js">
 44  *     fs.rename('/tmp/hello', '/tmp/world', function (err) {
 45  *       if (err) throw err;
 46  *       fs.stat('/tmp/world', function (err, stats) {
 47  *         if (err) throw err;
 48  *         console.log('stats: ' + JSON.stringify(stats));
 49  *       });
 50  *     });
 51  * </pre>
 52  * In busy processes, the programmer is <em>strongly encouraged</em> to use the
 53  * asynchronous versions of these calls. The synchronous versions will block
 54  * the entire process until they complete--halting all connections.
 55  */
 56
