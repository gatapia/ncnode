  4 /**
  5  * @fileoverview To use the HTTP server and client one must `require('http')`.
  6  *
  7  * The HTTP interfaces in Node are designed to support many features
  8  * of the protocol which have been traditionally difficult to use.
  9  * In particular, large, possibly chunk-encoded, messages. The interface is
 10  * careful to never buffer entire requests or responses--the
 11  * user is able to stream data.
 12  *
 13  * HTTP message headers are represented by an object like this:
 14  * <pre>
 15  *     { 'content-length': '123',
 16  *       'content-type': 'text/plain',
 17  *       'connection': 'keep-alive',
 18  *       'accept': '*/*' }
 19  * </pre>
 20  * Keys are lowercased. Values are not modified.
 21  *
 22  * In order to support the full spectrum of possible HTTP applications, Node's
 23  * HTTP API is very low-level. It deals with stream handling and message
 24  * parsing only. It parses a message into headers and body but it does not
 25  * parse the actual headers or the body.
 26  */
 27
