  8 /**
  9  * @fileoverview File I/O is provided by simple wrappers around standard POSIX functions.  To
 10  * use this module do `require('fs')`. All the methods have asynchronous and
 11  * synchronous forms.
 12  *
 13  * The asynchronous form always take a completion callback as its last argument.
 14  * The arguments passed to the completion callback depend on the method, but the
 15  * first argument is always reserved for an exception. If the operation was
 16  * completed successfully, then the first argument will be `null` or `undefined`.
 17  *
 18  * Here is an example of the asynchronous version:
 19  * <pre>
 20  *     var fs = require('fs');
 21  *
 22  *     fs.unlink('/tmp/hello', function (err) {
 23  *       if (err) throw err;
 24  *       console.log('successfully deleted /tmp/hello');
 25  *     });
 26  * </pre>
 27  * Here is the synchronous version:
 28  * <pre>
 29  *     var fs = require('fs');
 30  *
 31  *     fs.unlinkSync('/tmp/hello')
 32  *     console.log('successfully deleted /tmp/hello');
 33  * </pre>
 34  * With the asynchronous methods there is no guaranteed ordering. So the
 35  * following is prone to error:
 36  * <pre>
 37  *     fs.rename('/tmp/hello', '/tmp/world', function (err) {
 38  *       if (err) throw err;
 39  *       console.log('renamed complete');
 40  *     });
 41  *     fs.stat('/tmp/world', function (err, stats) {
 42  *       if (err) throw err;
 43  *       console.log('stats: ' + JSON.stringify(stats));
 44  *     });
 45  * </pre>
 46  * It could be that `fs.stat` is executed before `fs.rename`.
 47  * The correct way to do this is to chain the callbacks.
 48  * <pre>
 49  *     fs.rename('/tmp/hello', '/tmp/world', function (err) {
 50  *       if (err) throw err;
 51  *       fs.stat('/tmp/world', function (err, stats) {
 52  *         if (err) throw err;
 53  *         console.log('stats: ' + JSON.stringify(stats));
 54  *       });
 55  *     });
 56  * </pre>
 57  * In busy processes, the programmer is _strongly encouraged_ to use the
 58  * asynchronous versions of these calls. The synchronous versions will block
 59  * the entire process until they complete--halting all connections.
 60  */

 62 /**
 63  * Asynchronously reads the entire contents of a file. Example:
 64  * <pre>
 65  *     fs.readFile('/etc/passwd', function (err, data) {
 66  *       if (err) throw err;

 68  *     });
 69  * </pre>

 71  * contents of the file.

 75  * @param {*} encoding_
 76  * @return {*}
