  4 /**
  5  * @fileoverview Use `require('tls')` to access this module.
  6  *
  7  * The `tls` module uses OpenSSL to provide Transport Layer Security and/or
  8  * Secure Socket Layer: encrypted stream communication.
  9  *
 10  * TLS/SSL is a public/private key infrastructure. Each client and each
 11  * server must have a private key. A private key is created like this
 12  * <pre>
 13  *     openssl genrsa -out ryans-key.pem 1024
 14  * </pre>
 15  * All severs and some clients need to have a certificate. Certificates are public
 16  * keys signed by a Certificate Authority or self-signed. The first step to
 17  * getting a certificate is to create a "Certificate Signing Request" (CSR)
 18  * file. This is done with:
 19  * <pre>
 20  *     openssl req -new -key ryans-key.pem -out ryans-csr.pem
 21  * </pre>
 22  * To create a self-signed certificate with the CSR, do this:
 23  * <pre>
 24  *     openssl x509 -req -in ryans-csr.pem -signkey ryans-key.pem -out ryans-cert.pem
 25  * </pre>
 26  * Alternatively you can send the CSR to a Certificate Authority for signing.
 27  *
 28  * (TODO: docs on creating a CA, for now interested users should just look at
 29  * `test/fixtures/keys/Makefile` in the Node source code)
 30  */

 58 /**
 59  * Creates a new client connection to the given `port` and `host`. (If `host`
 60  * defaults to `localhost`.) `options` should be an object which specifies
 61  *
 62  *   - `key`: A string or `Buffer` containing the private key of the server in
 63  *     PEM format. (Required)
 64  *
 65  *   - `cert`: A string or `Buffer` containing the certificate key of the server in
 66  *     PEM format.
 67  *
 68  *   - `ca`: An array of strings or `Buffer`s of trusted certificates. If this is
 69  *     omitted several well known "root" CAs will be used, like VeriSign.
 70  *     These are used to authorize connections.
 71  *
 72  * `tls.connect()` returns a cleartext `CryptoStream` object.
 73  *
 74  * After the TLS/SSL handshake the `callback` is called. The `callback` will be
 75  * called no matter if the server's certificate was authorized or not. It is up
 76  * to the user to test `s.authorized` to see if the server certificate was
 77  * signed by one of the specified CAs. If `s.authorized === false` then the error
 78  * can be found in `s.authorizationError`.
 79  * @param {*} port /* host
 80  * @param {*} options
 81  * @param {*} cb */
 82  * @return {*}
 83  */
 84 node.tls.prototype.connect = function(port /* host, options, cb */) {
 85   return node.tls.core_.connect(port /* host, options, cb */);
 86 };
